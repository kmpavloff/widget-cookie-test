🔬 SameSite vs Partitioned Cookie Test SameSite=Strict CHIPS

Что тестируется: сайт example.localhost выставляет две куки:
  🔴 session_id — обычная кука с SameSite=Strict
  🟢 partitioned_id — partitioned кука (CHIPS) с SameSite=None; Partitioned

Ожидаемый результат:
  🔴 Обычная кука: в cross-site iframe не отправляется → сервер каждый раз создаёт новую
  🟢 Partitioned кука: каждый top-frame origin получает свою куку, но внутри одного origin — кука сохраняется при F5
Действия:
🖥️ Прямой доступ example.localhost напрямую
📦 Через site-a iframe на site-a.localhost
📦 Через site-b iframe на site-b.localhost
